# AstraScanner – 3D‑сканер на базе камеры Astra Pro

[![GitHub repo](https://img.shields.io/badge/GitHub-Klischa/AstraScanner-blue?logo=github)](https://github.com/Klischa/AstraScanner)

**AstraScanner** – это  приложение для 3D‑сканирования с использованием камеры **Orbbec Astra Pro** (или совместимых не проверял) и опционального поворотного стола. Программа позволяет захватывать, накапливать, фильтровать, регистрировать и экспортировать облака точек в популярные форматы: **PLY**, **STL**, **OBJ**. Поддерживается автоматическое сканирование с настраиваемым интервалом, калибровка камеры по шахматной доске, управление проектами и расширенное логирование.

---

## 🎯 Основные возможности

| Функция | Описание |
|---------|----------|
| **Захват видео** | Одновременный вывод RGB и depth‑потоков с камеры Astra Pro. |
| **Калибровка камеры** | По шахматной доске (настраиваемые размеры и количество углов). Результат сохраняется в `data/camera_calibration.xml`. |
| **Накопление облака точек** | В реальном времени, с возможностью паузы и сброса. |
| **Фильтрация облаков** | SOR (Statistical Outlier Removal), ROR (Radius Outlier Removal), Voxel Grid, «Magic Wand» (комбинация). |
| **Регистрация сканов (ICP)** | Pairwise ICP‑регистрация всех сканов проекта с настраиваемыми параметрами (max correspondence, итерации, финальная воксельная децимация). |
| **Ручное редактирование (лассо)** | Выделение произвольной области точек инструментом «петля» (freehand lasso), удаление / обрезка / инвертирование выделения, undo до 10 уровней. |
| **Автоматическое сканирование** | Задаётся интервал (сек) и количество сканов – идеально для поворотного стола. |
| **Управление сканами** | Сохранение текущего облака в список, очистка списка, экспорт выбранных или всех сканов в PLY. |
| **Экспорт** | PLY (бинарный, с цветом), STL (треугольная сетка), OBJ (геометрия + цвета). |
| **Проекты** | Сохранение и загрузка всех настроек и списка сканов (через `ProjectManager`). |
| **Логирование** | Расширенное логирование в файл `logs/scanner.log` (включение/выключение в настройках). |
| **GUI** | 6 вкладок (Главная, Сканирование, Калибровка, Обработка, Проект, Логи), 3D‑визуализатор на базе VTK + PCL. |

🚀 Использование

Первый запуск и калибровка

1. Подключите камеру Astra Pro к USB 3.0.
2. Запустите AstraScanner.exe.
3. Перейдите на вкладку «Калибровка», нажмите «Preview».
4. Поместите шахматную доску перед камерой, нажмите «Захватить кадр». Сделайте не менее 5 кадров с разных ракурсов.
5. Нажмите «Калибровать». При успехе интринсики сохранятся в data/camera_calibration.xml.

Простое сканирование

· «Preview» – только видеопотоки, накопление облака не происходит.
· «Scan» – начать накопление облака. Перемещайте камеру вокруг объекта.
· «Pause» – приостановить накопление.
· «Stop» – остановить сканирование.
· «Clear» – очистить текущее облако.

Работа со сканами (список)

· «Сохранить текущий скан» – копирует текущее облако в список сканов (вкладка «Обработка»).
· «Очистить список сканов» – удаляет все сохранённые сканы.
· Выделите один или несколько сканов (Ctrl+клик, Shift+клик).
· «Экспорт выбранных сканов (PLY)» – сохраняет выделенные сканы в выбранную папку (имена scan_1.ply, scan_2.ply …).
· «Экспорт всех сканов (PLY)» – сохраняет всё содержимое списка.

Объединение сканов (ICP‑регистрация)

1. Создайте или откройте проект, добавьте минимум 2 скана (вкладка «Проект» → «Добавить текущее облако как скан»).
2. Перейдите на вкладку «Обработка» → группа «Регистрация сканов (ICP)».
3. Настройте параметры:
   · **Max correspondence (м)** — максимальное расстояние поиска пар точек. Начните с 0.05 (5 см); для уже грубо выровненных сканов уменьшайте до 1–2 см.
   · **Итерации** — количество итераций ICP (по умолчанию 50).
   · **Финальный воксель (м)** — опциональная децимация результата (0 = без децимации). Полезно, т.к. N сканов дают N× плотность точек.
   · **Пропускать несошедшиеся** — если ICP не сошёлся на скане, пропустить его (иначе добавляется без трансформации).
4. Нажмите «Объединить все сканы проекта». Прогресс‑бар покажет ход, GUI остаётся отзывчивым (ICP выполняется в фоновом потоке).
5. После завершения результат отображается в 3D‑вьюаре. Нажмите «Сохранить результат в проект…» чтобы добавить объединённое облако как новый скан.

Автоматическое сканирование (для поворотного стола)

· На вкладке «Сканирование» в группе «Автоматическое сканирование»:
  · Включите чекбокс «Включить автосохранение».
  · Установите интервал (например, 1.7675 сек) и количество сканов (например, 8).
  · Нажмите «Сканировать с автосохранением».
· Каждые N секунд текущее облако будет сохраняться в список сканов, после чего облако очищается для следующего кадра.
· По достижении заданного количества сканов процесс автоматически останавливается.

Фильтрация облака

· SOR – удаляет выбросы (настраивается количество соседей и порог).
· ROR – удаляет точки с недостаточным количеством соседей в радиусе.
· Voxel Grid – уменьшает плотность облака.
· Magic Wand – быстрая очистка: сначала воксельный фильтр, затем SOR.

Ручное редактирование облака (лассо)

Инструмент «петля» позволяет вручную выделять и удалять произвольные области облака точек — полезно для удаления фона, стола, артефактов или любых нежелательных участков.

1. Загрузите облако (сканирование, загрузка из проекта или результат ICP‑мержа).
2. Перейдите на вкладку «Обработка» → группа «Ручное редактирование (лассо)».
3. Нажмите «Включить режим редактирования». Управление камерой через ЛКМ отключается; ПКМ и колесо мыши продолжают работать (вращение / масштаб).
4. **Зажмите ЛКМ** и обведите нужную область произвольным контуром (петлёй). При отпускании кнопки все точки, чья проекция на экран попадает внутрь контура, подсвечиваются красным.
5. Выполните операцию:
   · **Удалить выделенное** — удаляет подсвеченные точки из облака.
   · **Оставить выделенное** — удаляет всё *кроме* подсвеченных точек (crop).
   · **Инвертировать** — меняет выделение на противоположное.
   · **Снять выделение** — убирает подсветку без изменения облака.
   · **Отменить** — возвращает облако к предыдущему состоянию (undo, до 10 уровней).
6. Повторяйте шаги 4–5 для итеративной очистки.
7. Нажмите «Выключить режим редактирования» для возврата к стандартному управлению камерой.

> **Совет:** для больших облаков (>500K точек) выделение может занять ~1 секунду. Предварительно примените Voxel Grid для ускорения.

Экспорт итогового облака

· На вкладке «Сканирование» нажмите кнопку PLY, STL или OBJ.
· Укажите имя файла. Для STL/OBJ будет выполнена реконструкция поверхности (Poisson), для PLY – бинарное сохранение с цветом.

Настройки

· Меню «Настройки → Параметры» открывает диалог с 9 вкладками:
  · Камера – индекс, разрешение, прореживание точек.
  · Отображение – цветовая карта глубины, частота обновления GUI.
  · Облако – размер вокселя для даунсэмплинга, макс. количество точек.
  · Фильтры – параметры SOR, ROR, Voxel.
  · ICP – параметры регистрации (используется только при опциональном ICP).
  · Калибровка – размеры шахматной доски.
  · Сохранение – путь для экспорта, автосохранение проекта.
  · Логирование – включение записи в файл logs/scanner.log.
  · Поворотный стол – параметры автоматического сканирования.

🐛 Известные проблемы и их решение

| Проблема | Решение |
|----------|---------|
| Лог не пишется | Откройте «Настройки → Логирование» → поставьте галочку «Включить запись лога в файл» → OK. Или создайте папку `logs/` вручную и проверьте права на запись. |
| ICP не сходится | Увеличьте max correspondence distance (10 см) или уменьшите (1–2 см) для уже грубо выровненных сканов. Убедитесь, что сканы содержат перекрывающиеся области. |
| Лассо выделяет не те точки | Убедитесь, что камера повёрнута так, чтобы целевые точки не перекрывались другими. Используйте ПКМ/колесо для вращения, затем рисуйте лассо. |
| Камера не определяется | Проверьте драйверы OpenNI2, подключение к USB 3.0. |
| Медленное выделение лассо | Для облаков >500K точек проекция каждой точки на экран занимает время. Предварительно примените Voxel Grid. |

🏗️ Архитектура проекта

```
src/
├── main.cpp                         # Точка входа, установка message handler
├── gui/
│   ├── MainWindow.h / .cpp          # Главное окно: 6 вкладок, визуализатор, вся логика UI
│   └── LiveCloudWindow.h / .cpp     # Отдельное окно для live-просмотра облака
├── capture/
│   ├── AstraCamera.h / .cpp         # Обёртка над OpenNI2 для Astra Pro
│   └── CaptureWorker.h / .cpp       # QThread-воркер захвата кадров
├── calibration/
│   └── CameraCalibrator.h / .cpp    # Калибровка по шахматной доске (OpenCV)
├── filters/
│   └── PointCloudFilters.h / .cpp   # SOR, ROR, Voxel, ICP-merge, Poisson
├── editor/
│   └── PointCloudEditor.h / .cpp    # Лассо-выделение + удаление/crop/undo
├── project/
│   └── ProjectManager.h / .cpp      # Управление проектами (JSON + PLY)
├── export/
│   └── ExportManager.h / .cpp       # Экспорт в PLY / STL / OBJ
└── settings/
    └── SettingsManager.h / .cpp     # Персистентные настройки (QSettings)
```

Ключевые зависимости: **Qt 6** (GUI, потоки, настройки), **PCL 1.15** (облака точек, ICP, Poisson), **VTK** (3D‑визуализация), **OpenCV** (калибровка, обработка кадров), **OpenNI2** (драйвер камеры).
